# Kate Copilot v0.1.0 Release Notes

## Highlights
- Copilot-like inline completion for KDE Kate.
- Multi-line ghost text rendered by an overlay on `editorWidget()`.
- `Tab` full accept, `Esc` dismiss, next-word accept, and next-line accept.
- Rebindable Kate actions for inline-completion commands.
- OpenAI-compatible, Ollama, and GitHub Copilot OAuth providers.
- KWallet-backed API key and GitHub OAuth token storage.
- Prompt evaluation harness for HumanEval-Infilling and SAFIM.
- Gentoo live ebuild included in the repository.

## Shortcuts
- `Tab`: accept full suggestion
- `Ctrl+Alt+Shift+Right`: accept next word
- `Ctrl+Alt+Shift+L`: accept next line
- `Ctrl+Alt+Shift+Space`: trigger suggestion
- `Esc`: dismiss suggestion

All commands are also exposed as Kate actions for shortcut customization.

## Recommended Ollama preset
- Provider: `Ollama`
- Endpoint: `http://localhost:11434/v1/chat/completions`
- Model: `qwen3-coder-q4:latest`
- Prompt template: `FIM v3`

## Gentoo live ebuild
The ebuild lives at:

```text
packaging/gentoo/app-editors/kate-copilot/kate-copilot-9999.ebuild
```

Install through a local overlay:

```bash
sudo mkdir -p /var/db/repos/local/app-editors/kate-copilot
sudo cp -r packaging/gentoo/app-editors/kate-copilot/* \
  /var/db/repos/local/app-editors/kate-copilot/
cd /var/db/repos/local/app-editors/kate-copilot
sudo ebuild kate-copilot-9999.ebuild manifest
sudo emerge -av =app-editors/kate-copilot-9999
```

## Verification command

cmake -S . -B build -DBUILD_TESTING=ON

## Known limitations
- Overlay ghost text preserves the document buffer and paints above the editor; it cannot reserve virtual vertical space.
- KTextEditor framework-level virtual block support is the long-term route for exact Copilot-style multi-line layout.
- Copilot endpoints are unofficial for third-party clients and can change.
